# gpioctl-zsh v0.1.0 — Initial Open-Source Release

## English

This is the first open-source release of `gpioctl-zsh`, a from-scratch Linux
GPIO character-device control stack with an original versioned ioctl UAPI, an
extensible kernel HAL, a C library, and a CLI for one-shot, interactive, and
scripted control.

Highlights:

- exclusive per-file-descriptor leases, automatic cleanup, safe-state and
  IOPAD restoration;
- single-line operations, rollback-capable same-controller batches, edge
  events, bounded queues, statistics, and read-only sysfs observability;
- constrained IOPAD configuration protected by policy and `CAP_SYS_RAWIO`;
- a gpiolib backend, a Phytium IOPAD provider, and a fault-injectable mock;
- JSON Lines, dry-run validation, process-wide timeouts, REPL, and deterministic
  script execution;
- checked-in raw benchmark, static-analysis, stress, and hardware-smoke evidence.

Production and default-build artifacts do not copy, link, or depend on
libgpiod. An optional benchmark-only executable uses libgpiod's public API for
black-box comparison.

Hardware validation currently covers Phytium Pi on Linux
`6.6.63-phytium-embedded-v3.2`. Other SoCs, other kernel versions, 32-bit compat,
LOCKDEP runtime validation, and external-instrument electrical timing remain
unverified. The raw-MMIO lab is isolated, dangerous, and excluded from all
default build and installation paths.

target kernel from source.

## 中文

这是 `gpioctl-zsh` 的首次开源版本。项目从零实现 Linux GPIO 字符设备控制栈，
包含原创版本化 ioctl UAPI、可扩展内核 HAL、C 用户库，以及支持单命令、交互式和
脚本化控制的 CLI。

本版本已经在飞腾派与 Linux `6.6.63-phytium-embedded-v3.2` 上完成实板冒烟、故障
注入、并发/生命周期压力、静态分析和性能基准；完整原始数据保存在 `results/`。
当前没有宣称验证其他 SoC、其他内核、32 位 compat、LOCKDEP 运行态或外部仪器
电气时序。

生产与默认构建不复制、不链接、也不依赖 libgpiod；只有显式构建的黑盒性能基线
使用其公开 API。`lab/raw-mmio-lab/` 是不进入默认构建和安装路径的危险隔离实验，
不得在未证明硬件隔离的系统上执行写测试。

本 Release 只发布源码，不提供与特定内核绑定的预编译 `.ko` 或 CLI 二进制。

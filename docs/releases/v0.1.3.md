# gpioctl-zsh v0.1.3 — Reboot Persistence

## English

This patch release adds an explicit, opt-in boot lifecycle for the installed
GPIO stack. Installation starts the driver for the current session but leaves
boot autostart disabled. Users can later start it manually through systemd or
enable persistence with `gpioctl-zsh-autostart enable`.

Highlights:

- installs an idempotent `gpioctl-zsh.service` around the existing transactional
  load/unload scripts;
- provides `gpioctl-zsh-autostart enable|disable|status` without hiding the
  corresponding systemd commands;
- restores drivers, Device Tree overlay, udev permissions, and character devices
  after boot, but never restores previous GPIO output values;
- fails visibly after a kernel/module mismatch instead of compiling at boot or
  entering an unbounded restart loop;
- documents manual post-reboot startup, opt-in autostart, current-boot proof,
  logs, disable semantics, and kernel-upgrade recovery.

Hardware validation used a Phytium Pi running Linux
`6.6.63-phytium-embedded-v3.2`. After a real reboot, the service entered
`active (exited)` with `Result=success`, all six `/dev/gpioN_zsh` devices were
recreated, and `gpioctl_zsh list` enumerated all controllers. Shell parsing,
userspace self-tests, and systemd unit verification also passed.

No prebuilt kernel modules are attached. Build from source against the running
target kernel.

## 中文

本补丁版本为已安装的 GPIO 控制栈补齐显式、可选的开机生命周期。安装完成后驱动会
立即供当前会话使用，但默认不启用开机自启动；用户可以在重启后通过 systemd 手动
启动，或执行 `sudo gpioctl-zsh-autostart enable` 一次性开启以后每次开机启动。

自启动只恢复内核模块、设备树 overlay、udev 权限与字符设备，不保存或恢复 GPIO
输出电平。内核升级造成模块缺失或不兼容时，服务明确进入 failed，不在启动阶段自动
编译，也不无限重启。

本版本在 Linux `6.6.63-phytium-embedded-v3.2` 飞腾派上完成真实重启验证：服务
`Result=success`，六个 `/dev/gpioN_zsh` 均自动恢复，CLI 可正常枚举全部控制器；
Shell 语法、用户态自检与 systemd unit 验证亦通过。
